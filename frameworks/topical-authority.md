# Framework: Topical Authority & Semantic Content Networks

## Purpose
This framework provides a system for building topical authority through interconnected, entity-driven content clusters that search engines and AI systems can easily retrieve and understand.

## Core Principle
> **Isolated articles fail. Content must exist within a structured semantic network.**

---

## Phase 1: Topic Selection

### Step 1.1: Identify Core Topic
Choose a topic that:
- Aligns with business revenue goals
- Has sufficient search depth (minimum 20 related queries)
- You can realistically dominate (based on competitor analysis)

**Example:**
- ❌ Too broad: "SEO"
- ✅ Focused: "AI SEO for B2B SaaS"
- ❌ Too narrow: "Meta description optimization for AI"

### Step 1.2: Validate Topic Depth
Use Ahrefs/SEMrush to check:
- Number of related keywords: Minimum 50
- Parent topic coverage: Are there clear subtopics?
- Competitor topical coverage: Can you match/exceed their depth?

## Phase 3: Content Cluster Structure

### Step 3.1: Pillar Page Structure

**Pillar Page Template:**
```markdown
H1: [Core Topic] - Complete Guide [Year]

[Direct answer paragraph: 50-75 words defining topic]

## What Is [Core Topic]
- Definition with entity relationships
- Why it matters now (AI context)

## How [Core Topic] Works
- Mechanism explanation
- Visual diagram (optional)

## Key Components of [Core Topic]
### Component 1: [Subtopic]
- Definition
- Link to supporting article

### Component 2: [Subtopic]
- Definition
- Link to supporting article

[Continue for all subtopics]

## [Core Topic] Best Practices
- Actionable steps
- Examples

## Common Questions About [Core Topic]
- FAQ format (5-7 questions)

## Conclusion & Next Steps
```

### Step 3.2: Supporting Article Structure

**Supporting Article Template:**
```markdown
H1: [Specific Subtopic] - [Angle]

[Direct answer: 40-60 words]

## Context: Where This Fits in [Core Topic]
- Link to pillar page
- Relationship to broader topic

## [Subtopic] Explained
- Detailed definition
- Entity relationships

## How to [Action Related to Subtopic]
- Step-by-step
- Examples

## [Subtopic] Best Practices
- 3-5 actionable items

## Related Resources
- Links to other supporting articles
- Link back to pillar

## FAQ: [Subtopic]
- 2-3 specific questions
```

## Phase 4: Internal Linking Architecture

### Step 4.1: Link Types
| Link Type | Direction | Purpose |
|-----------|-----------|---------|
| Pillar → Supporting | Down | Direct users to deeper content |
| Supporting → Pillar | Up | Establish topic hierarchy |
| Supporting ↔ Supporting | Lateral | Connect related concepts |
| Contextual | In-content | Reinforce entity relationships |

### Step 4.2: Anchor Text Rules
| Rule | Example (Good) | Example (Bad) |
|------|----------------|---------------|
| Descriptive | "AI SEO content optimization guide" | "Click here" |
| Entity-focused | "Learn about LLM retrieval systems" | "This article" |
| Varied | Mix of exact, partial, branded | Same anchor everywhere |

### Step 4.3: Linking Density Guidelines
| Content Type | Internal Links (Min) | External Links (Min) |
|--------------|----------------------|----------------------|
| Pillar Page | 8-12 | 3-5 |
| Supporting Article | 4-6 | 1-3 |

## Phase 5: Semantic Language Optimization

### Step 5.1: Subject-Predicate-Object Structure
AI systems extract information best when content follows clear subject → predicate → object patterns.

Example:

| Subject | Predicate | Object |
|---------|-----------|--------|
| AI SEO | optimizes content for | LLM retrieval |
| Clearscope | analyzes | semantic relevance |
| Entity homes | establish | brand identity for AI |

Application:
- Write definitions in SPO format
- Use active voice (subject performs action)
- Avoid complex nested clauses

### Step 5.2: Entity Consistency
| Principle | Example |
|-----------|---------|
|Use same entity name throughout | Always "AI SEO" not mixing "GEO" / "AI optimization" |
| Define entities on first use | "Generative Engine Optimization (GEO) refers to..." |
| Maintain attributes consistently | If you say "AI SEO requires semantic clarity" don't later say "AI SEO needs clear writing" |

## Phase 7: Historical Trust Signals

### Step 7.1: Content Update Protocol
Search engines and AI systems value content with demonstrated historical performance.

| Action | Frequency | Impact |
|--------|-----------|--------|
| Minor updates (stats, examples) | Quarterly | Maintains freshness |
| Major refresh (new sections) | Annually | Shows continued investment |
| Date stamp update | When substantive changes made | Signals recency |

**Common Mistakes to Avoid**
| Mistake | Why It Fails | Correct Approach |
|---------|--------------|------------------|
| Publishing isolated articles | No semantic network for AI to follow | Always link to cluster |
| Shallow coverage of subtopics | AI sees incomplete expertise | Go deep on each subtopic |
| Inconsistent entity naming | Confuses AI entity recognition | Standardize terminology |
| Neglecting pillar page updates | Core authority decays | Refresh pillar quarterly |
| Weak internal linking | AI can't navigate topic relationships | Use descriptive anchors |
